

안녕하세요. 4조 발표 시작하겠습니다.  
이번 주 과제는 C언어로 SQL 처리기 구현하는 것이었으며,
과제의 핵심은 `입력 -> 파싱 -> 실행 -> 저장` 루프를 만드는 것이었습니다.
이제부터 저희 조의 경우 이 흐름을 어떻게 구현하였는지 보여드리겠습니다.

우선 저희는 필수 요구사항인 `INSERT`와 `SELECT`에 더해 `UPDATE`, `DELETE`를 구현하였습니다.
또한, CSV 헤더에 Primary Key, Unique Key, Not Null 정보를 함께 저장하여, 별도 CREATE TABLE 없이도 제약조건을 해석할 수 있도록 했습니다.
더불어 테이블을 메모리에 캐시하고 Primary Key 인덱스를 유지해 중복 검사를 더 빠르게 처리하도록 하였습니다.

구조도를 보며 설명드리겠습니다.

가장 먼저 `main`파일에서 SQL 파일을 한 글자씩 읽으면서
줄바꿈, 여러 줄 문장, 주석, 따옴표 안 문자를 구분해
입력을 하나의 SQL 문장 단위로 안전하게 분리합니다.

저희는 SQL 파일 안에 여러 문장이 들어오는 구조였기 때문에
먼저 `main`파일에서 문장 단위로 나누고,
그다음 단계들이 더 단순하게 동작하도록 설계하였습니다.

이후 `lexer`는 분리된 한 문장을 토큰으로 나눕니다.
현재 구현은 hand-written lexer 방식으로,
공백은 건너뛰고, 문자열은 작은따옴표 기준으로 읽고,
식별자, 숫자, 기호를 각각 규칙에 따라 분리합니다.

그다음 `parser`는 이 토큰 흐름이 문법에 맞는지 검사하고,
결과를 `Statement` 구조체로 정리합니다.
이번 과제 범위에 맞춰 필요한 정보만 구조체에 담는
단순한 hand-written parser로 구현하였습니다.

마지막으로 `executor`가 이 `Statement`를 실행하여
CSV 파일 데이터 변경 및 콘솔 출력을 수행합니다.

이것은 SQL 문자열을 바로 실행하지 않고, 먼저 구조화한 뒤 실행하는 구조입니다.
구조화를 통해 문법 오류를 더 명확하게 분리할 수 있고,
실행 단계에서 제약조건도 함께 검증할 수 있습니다.

이제 시연을 하겠습니다.

먼저 select문으로 전체 조회와 조건 조회를 보여줍니다.

이후 insert문으로 새 사용자를 추가하고, 바로 다시 조회해서 실제로 저장됐는지 확인합니다.

다음은 엣지 케이스 처리입니다.
해당 SQL문 안에는 조회 결과가 없는 경우, 값 안에 쉼표나 세미콜론이 들어가는 경우, Primary Key나 Unique Key, Not Null 위반, 그리고 대상이 없는 UPDATE나 DELETE 같은 예외 상황과 실행 결과 값을 볼 수 있는 SELECT문이 들어가 있습니다.
실행해보면 콘솔에 엣지 케이스에 대응하는 에러 메시지들과 엣지 케이스에서 시도한 내용이 반영되지 않은 SELECT RESULT 값이 출력된 것을 확인하실 수 있습니다.

update와 delete의 시연은 시간 관계상 생략하겠습니다.

마지막으로 협업 방식입니다.
우선 원하는 방식으로 각자 프롬프트를 만들어 총 세 개의 모델을 만들었습니다.
그 후, 각자 학습을 진행하였으며 학습한 내용을 바탕으로 코드 리뷰와 발표에 적합한 모델을 선정하였습니다.
이후 발표 준비, 추가 코드 리뷰, 리팩토링은 다 함께 모여서 진행하였습니다.

이상 발표 마치겠습니다. 감사합니다.
