# AGENTS.md

이 파일의 규칙은 이 저장소에서 작업하는 에이전트가 기본적으로 따라야 하는 저장소 전용 지침입니다.

## 코드 주석 규칙
- 주석은 한국어로 작성한다.
- 공개 헤더의 `struct`, `enum`, 공개 함수 선언에는 역할 설명 주석을 붙인다.
- source 파일의 함수는 처음 보는 사람이 흐름을 따라갈 수 있도록 목적 설명 주석을 붙인다.
- 내부 helper 함수도 이름만으로 의도가 충분히 드러나지 않으면 주석을 붙인다.
- 핵심 실행 흐름을 설명하는 주석에는 `🧭`를 사용한다.
- 중요한 개념, 설계 포인트, 확장 포인트를 설명하는 주석에는 `⭐`를 사용한다.
- 제약 조건, 주의점, 현재 버전의 한계를 설명하는 주석에는 `⚠️`를 사용한다.
- 자명한 한 줄 코드 설명이나 변수 대입을 그대로 읽는 수준의 주석은 쓰지 않는다.
- 주석은 구현 세부를 장황하게 반복하기보다 "왜 이 단계가 필요한지", "이 단계가 무엇으로 바뀌는지"를 설명한다.
- parser, binder, planner, executor, storage 경계에서는 입력과 출력이 무엇인지 드러나게 주석을 쓴다.

## 문서화 우선순위
- README와 공개 헤더만 읽어도 전체 구조를 이해할 수 있어야 한다.
- 복잡한 흐름은 `main -> parser -> binder -> planner -> executor -> storage` 순서로 설명한다.
- 새로운 기능을 추가할 때는 관련 구조체, 공개 함수, 핵심 분기점 주석도 함께 갱신한다.
